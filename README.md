Autonomous backup battery monitoring and management system for Red Sea reef aquariums.

## вљЎ Features

- **Battery monitoring** via INA226 (I2C) or Victron Blue Smart IP22 (BLE)
- **Instant outage detection** via 230V relay on GPIO
- **Progressive pump intensity** вЂ” configurable SoC-based levels (eco в†’ survival в†’ critical)
- **Per-device control** вЂ” set individual speeds for each ReefWave/ReefRun/ReefLED
- **3-level network failover** вЂ” direct reach в†’ Wi-Fi rejoin в†’ mirror hotspot
- **Push notifications** вЂ” via [ntfy.sh](https://ntfy.sh) (free, no account required)
- **4G LTE failover** вЂ” send alerts via USB modem (Huawei E3372) when Wi-Fi is down
- **Home Assistant integration** вЂ” MQTT auto-discovery with 10 sensors
- **Auto-detection** вЂ” scans your network for ReefBeat devices during setup
- **Bilingual** вЂ” French/English interface based on system locale

## рџљЂ Quick Install


The installer will:
1. Download the latest release
2. Install system and Python dependencies
3. Launch an interactive configuration wizard that:
   - Scans your network for ReefBeat devices

4. Install and enable the systemd service
5. Optionally start the service immediately

## рџ”„ Service management

The installer creates a systemd service called `reefbeat-energy-backup`:

```bash

# Start / stop / restart
sudo systemctl start reefbeat-energy-backup
sudo systemctl stop reefbeat-energy-backup
sudo systemctl restart reefbeat-energy-backup

# Live logs
sudo journalctl -u reefbeat-energy-backup -f

# Disable auto-start on boot
sudo systemctl disable reefbeat-energy-backup
```

To reconfigure at any time:

```bash
python3 ~/scripts/reefbeatEnergyBackup/configure.py
sudo systemctl restart reefbeat-energy-backup
```

## рџ“± Push notifications (ntfy.sh)

Receive alerts directly on your phone without Home Assistant, using the free [ntfy.sh](https://ntfy.sh) service.

**Setup:**
1. Install the ntfy app ([Android](https://play.google.com/store/apps/details?id=io.heckel.ntfy) / [iOS](https://apps.apple.com/app/ntfy/id1625396347))
2. Subscribe to your topic (e.g. `reefbeat-myreef-secret`)
3. Configure in `config.json` or via the wizard

**Notification events:**
- вљЎ Power outage detected (with SoC and estimated runtime)
- вњ… Power restored (with outage duration)
- рџџЎрџџ рџ”ґ Pump level changes (eco в†’ survival в†’ critical)
- рџљЁ Battery critically low (repeated alerts)
- рџ“Ў Network failover status

**Priorities:** outage = `high` (sound), critical = `urgent` (alarm), info = `default` (silent)

**Test from the command line:**

```bash
# Simple test
python3 test_notif.py

# Simulate specific alerts
python3 test_notif.py --type outage       # Power outage
python3 test_notif.py --type restored     # Power restored
python3 test_notif.py --type critical     # Critical battery (alarm)
python3 test_notif.py --type level        # Level change
python3 test_notif.py --type network      # Network failover

# Custom message
python3 test_notif.py --message "Hello from RPi"

# Force send via 4G LTE modem
python3 test_notif.py --type outage --lte

# Override priority
python3 test_notif.py --type test --priority urgent
```

Notifications are only sent automatically when a power outage is detected. On mains power, no notifications are sent (use `test_notif.py` for manual tests).

## рџ“¶ 4G LTE failover

When Wi-Fi and home router are down, notifications can be sent via a USB 4G modem.

**Recommended hardware:** [Huawei E3372h-320](https://www.amazon.fr/HUAWEI-51071SMK-Huawei-E3372h-320-LTE-Stick/dp/B085RDTZMP) (~40в‚¬)

<p align="center">
  <img src="docs/images/huawei-e3372h-320.png" alt="Huawei E3372h-320" width="300">
</p>

LTE Cat4 150 Mbps download, bands 1/3/7/8/20 (800/900/1800/2100/2600 MHz), plug-and-play HiLink mode.

The E3372h creates a virtual Ethernet interface (`eth1`) on the RPi вЂ” no driver or PPP configuration needed. Just plug it in with an active SIM card.

**How it works:**
1. Normal: notifications sent via Wi-Fi
2. Wi-Fi down: notifier detects failure, switches to LTE interface
3. Sends via `curl --interface eth1` to force traffic through the modem
4. HiLink web interface available at `http://192.168.8.1` for status/signal monitoring

## вЏ± Runtime estimate

The `runtime_h` sensor always shows the estimated autonomy вЂ” "if power cuts now, how long will the battery last?"

- **On battery:** calculated from real measured current (INA226), averaged over a sliding window.
- **On mains:** calculated from theoretical pump consumption at current intensity level, using the power tables from `power_estimation.py`.
